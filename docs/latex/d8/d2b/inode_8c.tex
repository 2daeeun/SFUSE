\texorpdfstring{$<$}{<} 블록 읽기/쓰기 (read\+\_\+block/write\+\_\+block) 디스크 읽기/쓰기 함수 (disk\+\_\+read/disk\+\_\+write)

새로운 아이노드를 기본 값으로 초기화한다.

이 함수는 파일 시스템 내에서 새로운 아이노드를 생성할 때 호출된다. 아이노드는 파일 또는 디렉터리와 같은 객체의 메타데이터를 관리하는 핵심적인 구조체로, 이 함수는 아이노드를 사용 가능한 기본값으로 설정하여 초기 상태를 정의한다.

세부적으로는 다음의 작업을 수행한다\+:
\begin{DoxyItemize}
\item 아이노드 메모리 영역을 모두 0으로 초기화한다.
\item 파일의 접근 권한(mode), 소유자(uid), 그룹(gid)을 설정한다.
\item 파일 크기를 0으로 설정한다 (새로 생성된 파일이므로 초기 크기는 0이다).
\item 접근(atime), 수정(mtime), 상태 변경(ctime) 시간을 현재 시각으로 설정한다.
\item 모든 Direct 블록 포인터를 0으로 초기화하여, 아직 데이터를 가리키지 않도록 설정한다.
\item Single indirect, Double indirect 블록 포인터 역시 0으로 설정하여, 추가적인 데이터 블록 참조가 없음을 나타낸다.
\item 링크 수(links)를 초기화한다. 만약 디렉터리라면 기본적으로 2개(자기 자신 \textquotesingle{}.\textquotesingle{}과 상위 디렉터리 \textquotesingle{}..\textquotesingle{}), 파일이라면 1개로 설정한다.
\end{DoxyItemize}

아이노드를 디스크에서 읽어 메모리에 로드한다. 

디스크에서 지정된 아이노드를 읽어 메모리에 로드한다.

파일 시스템에서 파일과 디렉터리의 메타데이터(크기, 권한, 소유자, 데이터 블록 위치 등)는 아이노드(inode)에 저장된다. 이 함수는 디스크에 저장된 아이노드 데이터를 읽어 메모리의 inode 구조체에 로드한다.

아이노드를 로드하는 과정은 다음과 같다\+:


\begin{DoxyEnumerate}
\item 아이노드 번호(ino)의 유효성을 검사한다.
\begin{DoxyItemize}
\item ino 번호는 0이 아니어야 하고, 슈퍼블록의 inodes\+\_\+count보다 작아야 한다.
\end{DoxyItemize}
\item 디스크 내의 아이노드 위치(offset)를 계산한다.
\begin{DoxyItemize}
\item 슈퍼블록의 inode\+\_\+table\+\_\+start는 아이노드 테이블의 시작 위치를 나타내는 블록 번호이다.
\item 아이노드의 크기(sizeof(\texorpdfstring{$\ast$}{*}inode))에 아이노드 번호를 곱해 정확한 위치를 계산한다.
\end{DoxyItemize}
\item 계산된 위치에서 아이노드 데이터를 읽어 inode 버퍼에 저장한다.
\begin{DoxyItemize}
\item 읽은 데이터의 크기가 아이노드 구조체 크기와 일치하지 않으면 입출력 오류(\+EIO)를 반환한다.
\end{DoxyItemize}
\end{DoxyEnumerate}

메모리에 저장된 아이노드를 디스크에 기록(동기화)한다. 

메모리에 저장된 아이노드 정보를 디스크에 기록한다.

파일 시스템은 아이노드를 메모리 내에서 수정한 후, 변경된 내용을 디스크에 영구적으로 저장해야 한다. 이 함수는 메모리 상의 아이노드를 디스크의 올바른 위치에 정확하게 기록하여 파일의 메타데이터가 유지되도록 한다.

아이노드를 디스크에 기록하는 과정은 다음과 같다\+:


\begin{DoxyEnumerate}
\item 아이노드 번호(ino)의 유효성을 검사한다.
\begin{DoxyItemize}
\item ino 번호는 반드시 0보다 크고 슈퍼블록의 총 아이노드 개수(inodes\+\_\+count)보다 작아야 한다.
\end{DoxyItemize}
\item 기록할 아이노드 데이터의 디스크 내 위치(offset)를 계산한다.
\begin{DoxyItemize}
\item 아이노드 테이블의 시작 위치(inode\+\_\+table\+\_\+start)를 블록 단위에서 바이트 단위로 변환하고, 여기에 아이노드 번호(ino)와 아이노드의 크기(sizeof(\texorpdfstring{$\ast$}{*}inode))를 곱하여 정확한 위치를 얻는다.
\end{DoxyItemize}
\item 메모리 상의 아이노드 데이터를 계산된 위치에 디스크로 기록한다.
\begin{DoxyItemize}
\item 기록 후, 실제로 기록된 바이트 수가 아이노드의 크기와 정확히 일치하는지 확인한다.
\item 기록된 바이트 수가 일치하지 않거나 오류가 발생하면 적절한 오류 코드를 반환한다.
\end{DoxyItemize}
\end{DoxyEnumerate}

논리 블록 번호(\+Logical Block Number, LBN)를 물리 블록 번호(\+Physical \+Block Number, PBN)로 변환한다. 

논리 블록 번호(\+Logical Block Number)를 물리 블록 번호(\+Physical Block \+Number)로 변환한다.

파일은 여러 블록에 걸쳐 데이터를 저장하며, 파일 내에서의 블록 위치는 논리적(\+LBN)으로 표현된다. 그러나 실제 디스크는 논리 블록을 직접적으로 알지 못하므로, 물리적인 블록 번호(\+PBN)로 변환해 접근해야 한다. 이 함수는 아이노드에 저장된 블록 주소를 참조하여 LBN을 PBN으로 변환해주는 역할을 수행한다.

LBN에서 PBN 변환 방법\+:
\begin{DoxyItemize}
\item \mbox{[}Direct Block 처리\mbox{]}
\begin{DoxyItemize}
\item LBN이 SFUSE\+\_\+\+NDIR\+\_\+\+BLOCKS(보통 12개)보다 작으면, direct 배열에서 직접 찾아 반환한다.
\end{DoxyItemize}
\item \mbox{[}Single Indirect Block 처리\mbox{]}
\begin{DoxyItemize}
\item Direct 블록으로 표현할 수 있는 범위를 넘어가면, Single indirect 블록에서 찾는다.
\item Single indirect 블록이 가리키는 블록 하나에는 여러 개의 블록 주소를 담을 수 있다.
\end{DoxyItemize}
\item \mbox{[}Double Indirect Block 처리\mbox{]}
\begin{DoxyItemize}
\item Single indirect 블록 범위도 초과하면, Double indirect 블록을 참조해야 한다.
\item Double indirect 블록은 두 단계의 포인터를 거쳐 주소를 참조한다.
\end{DoxyItemize}
\end{DoxyItemize}
